The Problem

MCP, function calling, and tool use standardised how AI agents act on the world. No equivalent exists for the other direction: humans telling agents what they want with structural precision.

A developer types "the hero font is too small." A client sends a Loom recording. A QA engineer files a ticket: "the text on the homepage? The big one?" Each message drops the DOM path, computed styles, component name, and spatial intent. Proprietary AI tools are building their own closed context capture. Without an open format now, structured human-to-AI communication locks into walled gardens.

What the PoC Implements

  • Bridge server + MCP tool handlers: WebSocket bridge delivers captures to AI tools, with handlers for get_session, get_voice_transcript, get_annotations, get_screenshot
  • Three output formats: plain text, JSON, Markdown with pluggable compiler architecture
  • Console and network capture: monkey-patches console.error, console.warn, fetch, XMLHttpRequest in the MAIN world via script injection (avoids chrome.debugger warning bar)
  • Screenshot intelligence: interest-scoring using frame differencing (30%), cursor dwell (25%), voice activity (35%) to auto-capture at moments of user attention
  • Annotation tools: circle, arrow, freehand, rectangle with scroll anchoring
  • On-device Whisper transcription: Transformers.js + Xenova/whisper-tiny.en, functional but needs production hardening
  • Cursor dwell tracking: temporal correlation to voice segments

Expected Outcomes

  • A privacy-first browser extension with local voice transcription, working across Chrome and Firefox, detecting React/Vue/Svelte components, capturing accessibility context
  • A versioned, documented open format specification for structured context capture, the input-side counterpart to MCP, that any AI tool, accessibility auditor, or UX researcher can implement
  • A standalone prompt compiler library (npm) that merges technical and human context with temporal correlation
  • An MCP-compatible bridge server published as a formal MCP tool for Claude Code, Aider, and others
  • A format designed for sharing between stakeholders. The collaborative sharing infrastructure (relay, viewer, team features) is the trajectory for a follow-up grant

If other tools adopt the format, PointDev succeeded. The extension is one implementation.
